How to Submit

Guidelines for contributing new articles

Contributing on Github

Contributing on GitHub is simple. Just follow these steps.

  1. Fork the repository.
  2. Create a branch called 'Article-title-of-article' or 'Change-to-title-of-article'.
  3. Make a change to existing HTML or add a new page on that new branch.
    • If you are creating a new article, please use 
      example.html
      • as a base for your new article, but copy it to a new file.
    • Title your new article 
      title-of-your-article.html
      .
    • Make sure your new article is stored in the root directory.
  4. Submit a pull request with your patches. The pull request should be titled the same as your branch.

Contributing outside of Github

You can submit articles in several ways:

  • You can go to please-dont.info/submit.html and submit your idea on the form there.
  • Send an email to 
    samathy[AT]sbarratt[DOT]co[DOT]uk
    .
  • Send a tweet or Direct Message to 
    @Samathy_Barratt
    .
  • Contact 
    @samathy
    on Telegram.

Articles should be around 200 to 300 words. They should provide links to relevent sources. If you contribute an article outside of Github, don't worry about formatting, I'll do that for you. The title of the article, should you choose to specify it yourself, should be in the form of a short sentence that continues on from "Please Don't .......".
